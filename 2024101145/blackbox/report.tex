\section{Detailed Test Case Design and Justification}
The following table lists the comprehensive set of test cases designed to validate the QuickCart API, covering valid requests, invalid inputs, missing fields, wrong data types, and boundary values.

\begin{enumerate}
    \item \textbf{Test Profile Retrieval (Valid)}
    \begin{itemize}
        \item \textbf{Input:} Valid roll number and user ID headers.
        \item \textbf{Expected Output:} 200 OK with profile JSON.
        \item \textbf{Justification:} Validates basic connectivity and authentication.
    \end{itemize}

    \item \textbf{Test Profile Name Length (Boundary - Min/Max)}
    \begin{itemize}
        \item \textbf{Input:} Name with 1 character and 51 characters.
        \item \textbf{Expected Output:} 400 Bad Request.
        \item \textbf{Justification:} Validates boundary constraints for the name field.
    \end{itemize}

    \item \textbf{Test Profile Phone Format (Invalid Input)}
    \begin{itemize}
        \item \textbf{Input:} Phone numbers with 3, 9, 11 digits, and non-numeric strings.
        \item \textbf{Expected Output:} 400 Bad Request.
        \item \textbf{Justification:} Ensures data integrity for contact information.
    \end{itemize}

    \item \textbf{Test Address Labels (Valid)}
    \begin{itemize}
        \item \textbf{Input:} Labels "HOME", "OFFICE", "OTHER".

    \end{itemize}

    \item \textbf{Test Address Pincode (Boundary/Invalid)}
    \begin{itemize}
